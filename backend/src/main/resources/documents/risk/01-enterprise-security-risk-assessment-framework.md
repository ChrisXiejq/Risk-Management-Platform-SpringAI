## Enterprise Security Risk Assessment Framework

This document provides a practical high-level framework for enterprise security risk assessment.

### 1. Risk discovery (what risks exist)
- Identify assets and business processes in scope.
- Identify threats and vulnerabilities relevant to the scope.
- Consider threat actors, attack vectors, and existing security weaknesses.
- Map risks to a risk taxonomy (e.g., confidentiality / integrity / availability, or threats/vulnerabilities).

### 2. Asset assessment (how bad and how likely)
- Assess impact: potential business harm (financial loss, operational disruption, reputational damage, regulatory exposure).
- Assess likelihood: based on exposure, current controls, threat intelligence, and historical incidents.
- Produce a qualitative risk level (e.g., Low/Medium/High) and document assumptions.

### 3. Governance decision (what to do)
- Choose mitigations: preventive, detective, responsive, and recovery actions.
- Align controls with recognized standards/frameworks (e.g., ISO/IEC 27001, NIST CSF, OWASP).
- Create an implementation roadmap with ownership and timelines.
- Define monitoring metrics and review cadence.

### Common uncertainties
- Incomplete asset inventory, missing control maturity evidence, and uncertain threat assumptions are typical sources of uncertainty.
- When evidence is missing, use web search / additional data collection before finalizing rankings.
