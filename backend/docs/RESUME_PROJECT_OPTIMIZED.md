# 专利成果转化智能 Agent 平台 — 简历项目描述（优化版）

> 2025.12–2026.03，核心成员。面向**后端 + Agent 工程师**岗位，优先突出 Agent 设计与实现，RAG 置于后段。

---

**2025.12 – 2026.03  专利成果转化智能 Agent 平台 | 核心成员**

基于 Spring Boot 3 + Spring AI 的专利成果转化智能平台，以多智能体图编排与 ReAct 执行为核心，面向企业/高校/个人提供专利转化智能查询与成果调研，解决专利技术理解成本高、转化决策难等问题。

1. **多智能体图编排与任务分解**：基于 LangGraph4j 实现 Plan-and-Execute 图编排，由 Planner 根据用户请求生成「检索 → 价值评估 → 建议 → 综合」等任务列表，Executor 对每个子任务执行 ReAct 循环（思考-行动-观察）；通过 maxSteps 与 recursionLimit 控制步数与递归深度，并对简单问候、自我介绍等做短路路由，减少无效 LLM 调用与成本。

2. **RePlan 与执行鲁棒性**：在检索失败或结果不足时触发 RePlan，支持整计划重算与仅重规划剩余任务两种模式，并与环境变化检测联动（如专利失效、接口异常）；当知识库无相关内容时自动引导检索专家调用 Web Search 补足，使复杂查询与异常场景下的完成率与稳定性明显提升。

3. **反思与策略注入**：设计并实现「步骤成败 → 规则归纳 → 写入记忆 → 注入规划与 prompt」的闭环：在 RePlan 或步骤结束后对结果进行反思，将成功/失败经验归纳为简短可复用规则写入记忆，并在后续规划与专家系统提示中注入「Lessons learned」，使 Agent 在相似场景下减少重复错误、提升决策一致性。

4. **三层记忆架构与按需拉取**：设计并实现 Working → Experiential → Semantic 三层记忆：Working 采用滑动窗口与重要性剪枝管理短期上下文，Experiential 通过递归摘要与 decay 权重存储事件摘要与用户意图，Semantic 经 NLI 冲突检测后写入长期事实与偏好；记忆不由系统自动注入，而是由 Agent 通过工具按需拉取，在控制 token 消耗的同时减少无关历史对当前任务的干扰。

5. **Agent Skills 与集中式工具**：引入 Agent Skills 架构（SKILL.md + SkillRegistry），将检索/价值评估/建议等专家指令外部化为可配置技能文件，支持按任务加载与渐进式披露，便于提示词迭代与多技能扩展；基于单例与 Spring Bean 实现集中式工具注册，统一管理专利热度查询、Web Search、记忆检索等 Tool，部分能力采用单独部署 + MCP 协议集成，与主服务解耦，保障稳定性与安全性。

6. **业务中台与基础能力**：负责基于 Spring Boot 构建业务中台，实现专利问卷调研、成果转化查询等核心能力；设计并实现 JWT 鉴权、黑名单与 Token 自动刷新机制，解决多端登录下的鉴权与续期问题；引入 Redis 做热点数据缓存并配置过期策略，有效降低数据库压力与接口响应时间。

7. **RAG 检索链路与召回优化**：负责文档语义切分与 Token 回退策略，在保证长文信息保留率的前提下控制单块长度；实现向量检索（PgVector）+ BM25 的混合召回、RRF 融合与 Cohere Rerank，并引入 HyDE（用 LLM 生成假设性答案再做向量检索）与多查询扩展（MQE）优化原始 query，为 Agent 检索子任务提供更相关上下文。

8. **CRAG 门控与 GraphRAG 风格实体召回**：在检索链后增加 CRAG 层，对召回文档做质量评分并按置信度三分支（高置信直接使用、模糊时截断、低置信时弃用并触发 Web 兜底），与空上下文策略统一成一套门控；同时构建实体-文档索引，查询时按实体召回并与主检索结果 RRF 融合。在 RAGAS 自动化评估下，Recall@20 从 62% 提升至 80%，Top@1 准确率提升约 10%。

**技术栈**：Spring AI，Agent（ReAct，LangGraph4j，Plan-and-Execute），RAG（PgVector，CRAG / HyDE / GraphRAG），Spring Boot，MySQL，Redis，MCP。
